==================================================
PRIMARY GOAL
==================================================
Make the project interactive so I can run simple commands like:

- python alexnet.py
- python resnet18.py
- python vgg16.py
- python convnextv2.py

and then the script will interactively ask me what I want to do, such as:
- which task
- which input representation
- train or test or explain
- which checkpoint to use
- which device to use
- whether to use the default recommended config
- whether to run raw RGB or masked/paper-style path

The script should then route to the correct existing pipeline logic automatically.

==================================================
DESIGN PRINCIPLE
==================================================
Do NOT duplicate training logic in each top-level script.

Instead:
- create a reusable interactive launcher/helper layer
- keep the actual training/evaluation/explainability logic in the existing modules
- top-level scripts should act as friendly wrappers over the current config-driven pipeline

The goal is:
- preserve maintainability
- preserve existing configs
- preserve existing output conventions
- reduce user memory burden

==================================================
USER EXPERIENCE GOAL
==================================================
I want to be able to do things like:

   python alexnet.py

   - Choose action: train / eval / explain / predict

2. Run:
   python convnextv2.py

==================================================
REQUIRED INTERACTIVE BEHAVIOR
==================================================
Each model script must guide the user through choices interactively.

At minimum support these actions:
- train
- eval
- explain
- predict

For train:
- ask task
- ask preprocessing mode
- ask device
- optionally ask whether to use default recommended config or customize basic parameters

For eval:
- ask task
- ask preprocessing mode if needed to identify config
- ask checkpoint path OR present a numbered list of matching checkpoints found under models/checkpoints/
- ask split to evaluate: val/test
- ask device

For explain:
- ask task
- ask preprocessing mode
- ask checkpoint path OR numbered selection
- ask predictions CSV path OR auto-detect from outputs/predictions/<experiment>/
- ask device

For predict:
- ask for image path
- ask checkpoint path OR numbered selection
- ask task/config context
- ask device

==================================================
SMART DEFAULTS REQUIREMENT
==================================================
The interactive scripts must have sensible defaults.

Examples:
- default task for baseline models: pattern3
- default preprocessing: raw_rgb
- default device: cuda if available, else cpu
- default split for eval: test
- default checkpoint selection: best.pt from the most relevant experiment folder
- default config selection should follow the existing configs if they exist

The user should be able to simply press Enter to accept sensible defaults.

==================================================
CONFIG MAPPING REQUIREMENT
==================================================

  -> configs/train_alexnet_raw.yaml

  -> configs/train_alexnet_paper.yaml

- ResNet18 + pattern3 + raw_rgb + train
  -> configs/train_resnet18_raw.yaml

- VGG16 + pattern3 + raw_rgb + train

- ConvNeXtV2 + pattern3 + masked_highlight_proxy + train
  -> configs/train_convnextv2_paper.yaml

If a chosen combination does not yet have a config:
- say so clearly
- do not crash
- optionally suggest the closest available config
