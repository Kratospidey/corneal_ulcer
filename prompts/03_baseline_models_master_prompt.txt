You are a senior medical computer-vision engineer and ML research implementation lead.

You are working on a corneal ulcer classification project based on the SUSTech-SYSU corneal ulcer dataset.

This project is classification-first. The EDA and dataset-audit phases have already been developed to establish what the dataset actually contains, how labels are structured, what leakage risks exist, and which tasks are feasible.

Your task now is to build the ACTUAL baseline classification training, evaluation, reporting, and explainability pipeline using three classical CNN baselines before any ConvNeXtV2 work begins.

==================================================
PRIMARY GOAL
==================================================
Generate a complete, reproducible, leakage-aware Python classification pipeline that trains, evaluates, compares, and analyzes these three baseline models:

1. ResNet-18
2. AlexNet
3. VGG16

These baselines must be fully working end-to-end with:
- safe data loading
- split handling
- training
- validation
- test evaluation
- saved checkpoints
- metrics
- confusion matrices
- ROC/PR curves where valid
- per-image prediction exports
- explainability outputs
- summary reports

The purpose of this phase is to establish reliable baseline performance and a trustworthy reference point before later moving to ConvNeXtV2.

PROJECT PHILOSOPHY
==================================================
This phase exists to answer:
- can the full classification stack run correctly?
- what do simple/classic CNN baselines achieve?
- which task is the most stable and defensible?
- what failure modes appear before moving to a stronger architecture?
- how much benefit, if any, comes from paper-inspired preprocessing vs raw RGB?

Do not jump ahead to ConvNeXtV2 in this phase.

Do the baselines properly first.

==================================================
EXPERIMENT PRIORITY RULE
==================================================
You must prioritize experiments in this order:

Stage A:
- Choose the single most stable, leakage-safe confirmed task from the earlier EDA.
- Train and evaluate all three baseline models on that task using raw RGB inputs.

Stage B:
- For the same task, compare raw RGB vs paper-inspired preprocessing.
- This should be done for all three models if practical.
- If runtime or project complexity becomes excessive, prioritize:
  1. ResNet18 raw vs paper
  2. VGG16 raw vs paper
  3. AlexNet raw vs paper

Stage C:
- If the dataset and earlier analysis support a second task cleanly, extend the best-performing or most stable baseline configuration to that second task.
- Do not expand to extra tasks unless justified.

This project phase is about reliable baselines, not exhaustive combinatorial search.
